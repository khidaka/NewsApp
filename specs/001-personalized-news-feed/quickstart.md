## ビルド & 実行

1. Xcode でターゲット `NewsApp` を選択
2. Simulator または接続済み iPhone を選択
3. `Cmd + R` でビルド・実行

---

## 初期設定（初回起動時）

### ニュースソースを追加する

1. アプリ起動後、右上の設定アイコン → **ニュースソース** を開く
2. **＋ ボタン** をタップ
3. RSS / Atom フィードの URL を入力（例: `https://www3.nhk.or.jp/rss/news/cat0.xml`）
4. **追加** をタップ → 一覧に表示されることを確認

### Obsidian Vault を設定する（任意）

1. 設定 → **Obsidian Vault** をタップ
2. **フォルダを選択** → ファイルピッカーで iCloud Drive 上の Obsidian Vault フォルダを選択
3. 「アクセスを許可」 → 選択したパスが表示されることを確認

## 動作検証（P1: スワイプ閲覧・共有）

```
1. ホーム画面でニュースカードが表示されることを確認
2. カードを右スワイプ → iOS 共有シートが表示されることを確認
3. 共有シートで Readwise Reader を選択 → 記事が追加されることを確認
4. カードを左スワイプ → カードが消えることを確認
5. すべてカードをスワイプ後 → リロードボタンをタップ → 新しいカードが取得されることを確認
```

## 動作検証（P2: ニュースソース管理）

```
1. 設定でソースを1件追加 → リロード → 追加ソースの記事が表示されることを確認
2. 設定でソースを削除 → リロード → 削除ソースの記事が表示されないことを確認
```

## 動作検証（P3: パーソナライズ）

```
1. 特定トピック（例: AI）の記事を10件以上左スワイプ（興味なし）
2. リロード → AI 関連記事の表示頻度が減少していることを確認
3. Obsidian Vault を設定後リロード → Vault 内頻出キーワード関連記事が上位に配置されることを確認
```

## 動作検証（Undo）

```
1. 任意のカードをスワイプ
2. Undo ボタン（または画面シェイク）をタップ → 直前のカードが先頭に戻ることを確認
3. さらに別のカードをスワイプ後に Undo → 最後にスワイプした1件のみ戻ることを確認
```

## テスト実行

```bash
# Xcode のテストナビゲーター、または
Cmd + U  # 全テスト実行
```

主なテストターゲット：

| テスト | 内容 |
|--------|------|
| `PersonalizationServiceTests` | スコアリングアルゴリズムの正確性 |
| `RSSParserServiceTests` | RSS 2.0 / Atom 解析の正確性 |
| `ObsidianReaderServiceTests` | Markdown キーワード抽出 |
| `NewsCollectorIntegrationTests` | 重複除去・50件制限の統合テスト |

## トラブルシューティング

| 症状 | 対処 |
|------|------|
| カードが表示されない | ソースが未登録。設定から RSS URL を追加する |
| Obsidian キーワードが効いていない | 設定 → Obsidian Vault が選択されているか確認。アプリを再起動して再スキャンを実行 |
| 共有シートに Readwise Reader が表示されない | Readwise Reader アプリがインストールされているか確認 |
| バックグラウンド取得が遅い | iOS の省電力制御によるもの。手動リロードで代替 |
