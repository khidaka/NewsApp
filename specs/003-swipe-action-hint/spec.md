# Feature Specification: スワイプアクションヒント

**Feature Branch**: `003-swipe-action-hint`

**Created**: 2026-05-29

**Status**: Draft

**Input**: 各方向のスワイプの途中に、今どの操作をしようとしているのかわかるようにしてほしい

## User Scenarios & Testing *(mandatory)*

### User Story 1 - スワイプ中にアクションの種類がわかる (Priority: P1)

ユーザーがカードをドラッグしている最中に、現在の方向に対応するアクション名またはアイコンが
カード上に表示される。ドラッグを離すと確定・キャンセルを問わずヒントは消える。

**Why this priority**: これだけでユーザーが「何が起きるか」を把握できる。誤操作の防止と
ディスカバリビリティ向上の核心機能。

**Independent Test**: カードを右方向にドラッグすると「共有」の表示が現れ、左にドラッグすると
「興味なし」、上にドラッグすると「スキップ」の表示が現れることをSimulatorで確認できる。

**Acceptance Scenarios**:

1. **Given** ニュースカードが表示されている状態で、**When** ユーザーが右方向へドラッグを開始する、**Then** カード上に「共有」を示すヒント（アイコンまたはラベル）が表示される
2. **Given** ニュースカードが表示されている状態で、**When** ユーザーが左方向へドラッグを開始する、**Then** カード上に「興味なし」を示すヒント（アイコンまたはラベル）が表示される
3. **Given** ニュースカードが表示されている状態で、**When** ユーザーが上方向へドラッグを開始する、**Then** カード上に「スキップ」を示すヒント（アイコンまたはラベル）が表示される
4. **Given** ヒントが表示されている状態で、**When** ユーザーがドラッグを途中でキャンセル（閾値未満で離す）する、**Then** ヒントが消えてカードが元の位置に戻る
5. **Given** ヒントが表示されている状態で、**When** ユーザーがドラッグを完了（閾値以上で離す）する、**Then** カードがフライアウトし、ヒントもカードとともに消える

---

### User Story 2 - ヒントの視認性と強調 (Priority: P2)

ドラッグ距離が確定閾値に近づくにつれ、ヒントの表示が強調されることで「もうすぐ確定する」
ことがわかる。

**Why this priority**: P1 だけでも機能するが、閾値へのフィードバックがあると誤操作がさらに
減る。P1 の動作確認後に追加できる。

**Independent Test**: ドラッグ距離が確定閾値の 50% 未満のときヒントが薄く、75% 以上で
はっきり表示されることを確認できる。

**Acceptance Scenarios**:

1. **Given** カードをドラッグしている状態で、**When** ドラッグ距離が確定閾値の 50% 未満の場合、**Then** ヒントは半透明（目立たない）で表示される
2. **Given** カードをドラッグしている状態で、**When** ドラッグ距離が確定閾値の 75% 以上の場合、**Then** ヒントは不透明（はっきり）で表示される

---

### Edge Cases

- ドラッグ距離がごくわずか（5pt 未満）の場合、ヒントは表示しない（誤タッチとの区別）
- 斜め方向にドラッグした場合、水平・垂直どちらが優勢かで判定し、対応するヒントのみを表示する（両方同時には表示しない）
- ヒント表示中に別の指でタップするなど予期しない操作をした場合、ヒントは非表示になりカードは元の位置に戻る

## Requirements *(mandatory)*

### Functional Requirements

- **FR-001**: ドラッグ中、移動方向（右・左・上）に対応するアクションのヒントがカード上に表示されなければならない
- **FR-002**: ヒントは右方向で「共有」、左方向で「興味なし」、上方向で「スキップ」をそれぞれ示さなければならない
- **FR-003**: ドラッグ距離が 5pt 未満の場合、ヒントを表示してはならない
- **FR-004**: 斜め方向の場合、水平・垂直のどちらが優勢かのみに基づきヒントを1種類だけ表示しなければならない
- **FR-005**: ドラッグをキャンセルしたとき（閾値未満で離した場合）、ヒントは消えなければならない
- **FR-006**: ドラッグ距離に応じてヒントの不透明度が変化しなければならない（距離が増すほど濃くなる）
- **FR-007**: ヒントはカード本体のコンテンツ（タイトル・概要）の視認性を著しく損ねてはならない

## Success Criteria *(mandatory)*

### Measurable Outcomes

- **SC-001**: ドラッグ開始から 100ms 以内にヒントが表示される（応答性の確認）
- **SC-002**: ドラッグキャンセル後 200ms 以内にヒントが消える
- **SC-003**: ヒント表示中もカードのタイトルと概要が読める状態を維持する（ヒントがコンテンツを完全に覆わない）

## Assumptions

- ヒントはカードの中央上部または左右端付近に表示する（実装フェーズでデザインを決定）
- ヒントはアイコン（SF Symbols）＋短いラベルテキストの組み合わせを基本とする
- 右：`square.and.arrow.up`（共有）、左：`hand.thumbsdown`（興味なし）、上：`forward.end`（スキップ）
- カードの不透明度変化（既存の `opacity` アニメーション）とは独立して制御する
- ヒントの不透明度は `max(0, distance / threshold)` を基本計算式として実装する
