# Quickstart Validation: スキップ記事の永続的非表示

**Branch**: `006-skip-persist` | **Date**: 2026-05-29

## ビルド & テスト

```bash
# プロジェクト再生成
xcodegen generate

# ビルド確認
xcodebuild -scheme NewsApp \
  -destination 'platform=iOS Simulator,name=iPhone Air,OS=26.4.1' \
  build

# テスト実行（36件 → フィーチャー完成後は増加予定）
xcodebuild test -scheme NewsApp \
  -destination 'platform=iOS Simulator,name=iPhone Air,OS=26.4.1'
```

## 手動検証チェックリスト

### SC-001: スキップ記事の永続的非表示

1. シミュレータでアプリを起動する
2. フィードに記事が表示されていることを確認
3. 記事を上スワイプ（スキップ）する → 記事がフィードから消えることを確認
4. アプリをバックグラウンドに移動し、シミュレータで「Terminate」してアプリを終了
5. アプリを再起動する
6. スキップした記事がフィードに表示されないことを確認 ✓

### SC-002: スキップ後の遅延なし

1. 記事をスキップする
2. 次のカードが即座に表示されることを確認（待ち時間を感じないこと）✓

### SC-003: アンドゥ操作

1. 記事をスキップする → ツールバーに「取り消し」ボタンが表示されることを確認
2. 「取り消し」をタップする → スキップした記事がフィードに戻ることを確認
3. アプリを再起動する → アンドゥで戻した記事が引き続き表示されることを確認 ✓

### SC-004: 大量スキップ時のパフォーマンス（開発環境での確認）

1. テストデータで多数の記事をスキップ済み状態に設定（SwiftData Debug ツールなど）
2. フィードの表示速度に体感できる遅延がないことを確認 ✓

## 回帰テスト確認項目

- 右スワイプ（共有）が引き続き機能すること
- 左スワイプ（興味なし）が引き続き機能すること
- スワイプ後のアンドゥが引き続き機能すること
- おすすめソース（Feature 005）の表示が崩れていないこと
