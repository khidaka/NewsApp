# Feature Specification: スキップ記事の永続的非表示

**Feature Branch**: `006-skip-persist`

**Created**: 2026-05-29

**Status**: Draft

**Input**: User description: "スキップした記事はもう出てこないようにしたい"

## User Scenarios & Testing *(mandatory)*

### User Story 1 - スキップした記事が再起動後も非表示になる (Priority: P1)

ユーザーがスワイプ操作で記事をスキップすると、その記事はアプリを再起動しても二度とフィードに表示されない。

**Why this priority**: 現状はセッション中のみ非表示で、再起動すると同じ記事が繰り返し表示されてしまう。永続化することがこのフィーチャーの核心的な価値。

**Independent Test**: 記事をスキップしてアプリを再起動し、スキップした記事がフィードに表示されないことを確認することで単独でテスト可能。

**Acceptance Scenarios**:

1. **Given** ユーザーがフィードで記事を表示している, **When** 記事をスキップ操作する, **Then** その記事はフィードから即座に消える
2. **Given** ユーザーが記事をスキップした後にアプリを終了して再起動する, **When** フィードを開く, **Then** スキップ済みの記事は表示されない
3. **Given** 同じ記事が複数のフィード更新で再取得される, **When** フィードを表示する, **Then** スキップ済みの記事は表示されない

---

### User Story 2 - スキップ直後のアンドゥ操作が引き続き機能する (Priority: P2)

ユーザーは記事をスキップした直後に、その操作をアンドゥして記事を再度フィードに表示させることができる。

**Why this priority**: 誤操作への対応として現行のアンドゥ機能が維持されることが重要。ただし永続化後はアンドゥ後に再スキップが可能であることも確認が必要。

**Independent Test**: 記事をスキップしてすぐにアンドゥ操作を行い、記事がフィードに戻ることを確認することで単独でテスト可能。

**Acceptance Scenarios**:

1. **Given** ユーザーが記事をスキップした直後, **When** アンドゥ操作を行う, **Then** スキップした記事がフィードに戻り、スキップ記録も取り消される
2. **Given** ユーザーがアンドゥで記事を戻した後, **When** アプリを再起動する, **Then** アンドゥで戻した記事は再び表示される（スキップ記録が消えているため）

---

### Edge Cases

- すでにスキップ済みの記事 URL が RSS フィード更新で再取得された場合、フィルタリングが正しく行われるか？
- スキップ記事の件数が非常に多くなった場合（数百〜数千件）でも、フィードの表示速度に影響が出ないか？
- 同一記事が URL の細かい差異（クエリパラメータなど）で複数バージョン存在する場合はどう扱うか？
- アンドゥ後にアプリをクラッシュ/強制終了した場合、スキップ記録の状態はどうなるか？

## Requirements *(mandatory)*

### Functional Requirements

- **FR-001**: システムは記事のスキップ操作が行われた際に、当該記事を永続的な非表示リストに追加しなければならない
- **FR-002**: システムはフィードを表示する際に、永続的な非表示リストに含まれる記事を除外しなければならない
- **FR-003**: 永続的な非表示リストはアプリを再起動しても保持されなければならない
- **FR-004**: ユーザーはスキップ操作の直後にアンドゥ操作で記事を非表示リストから削除できなければならない
- **FR-005**: フィード更新（RSS の再取得）後も、スキップ済みの記事は表示されてはならない
- **FR-006**: スキップ記事を後から復元する手段はアンドゥ操作のみとし、管理画面（一覧・個別復元）は提供しない

### Key Entities

- **スキップ記録 (SkippedArticle)**: スキップされた記事を一意に識別する情報（記事 URL）と、スキップされた日時
- **フィード (Feed)**: 表示対象の記事リスト。スキップ記録を参照して非表示記事を除外した状態で提供される

## Success Criteria *(mandatory)*

### Measurable Outcomes

- **SC-001**: スキップした記事は、アプリ再起動後も含めてフィードに一切表示されない（再現率 0%）
- **SC-002**: スキップ操作後のフィード表示に、スキップ前と比較して体感できる遅延が生じない（ユーザーが待ち時間を感じない）
- **SC-003**: スキップ直後のアンドゥ操作で、記事が確実にフィードに戻る（成功率 100%）
- **SC-004**: スキップ記録が 1000 件以上存在する状態でも、フィードの読み込みに体感できる遅延が生じない

## Assumptions

- 記事の同一性は URL で判定する（既存の実装方針と整合）
- スキップ記録の管理画面（一覧・復元）は本フィーチャーのスコープ外とする（FR-006 が確認されるまでの暫定）
- アンドゥはスキップ直後の 1 操作のみに限定する（現行の Undo 設計を踏襲）
- スキップ記録は端末ローカルに保存し、クラウド同期は対象外
- 既存のセッション内スキップ変数は永続化実装に置き換えるか、両立させるかは実装フェーズで決定
